# Purpose

Use this skill for onboarding and activation flows in a SaaS or product platform.

This skill is for:
- first-run experiences
- setup wizards
- profile completion flows
- activation sequences
- guided next-step flows
- product entry experiences after sign-up
- reducing friction before first value

# Use this skill when

Use this skill when the request involves:
- creating onboarding steps
- improving activation flows
- reducing setup confusion
- clarifying first actions after sign-up
- improving profile or workspace setup UX
- making onboarding more premium and efficient

# Do not use this skill when

Do not use this skill for:
- generic forms with no onboarding significance
- billing-only flows
- public landing pages
- auth-only entry flows unless they continue into activation

# Onboarding philosophy

Onboarding should move the user to first value quickly.

It should answer:
- what should I do first
- why does this matter
- how many steps are left
- what happens next
- what can I skip or return to later

A good onboarding flow reduces uncertainty and accelerates activation.

# Workflow

## 1. Define first-value target

Determine:
- what “activated” means
- what the user must complete
- what is optional
- what can be deferred
- what blocks useful entry into the product
- where users are most likely to drop off

## 2. Structure the onboarding journey

Prefer:
- clear progression
- manageable step count
- concise step labels
- meaningful progress feedback
- optional steps clearly marked
- reduced unnecessary friction

## 3. Improve instructional clarity

Ensure:
- each step explains what matters
- the user understands why the step exists
- the next action is obvious
- the copy reduces uncertainty
- transitions between steps feel intentional

## 4. Handle onboarding states and exits

Check:
- skip behavior
- save-and-continue behavior
- return-later behavior
- validation or setup failure handling
- empty/default states after onboarding
- post-onboarding success transition

## 5. Review activation quality

Ask:
- does this move the user toward value fast
- is anything unnecessary
- are there too many fields or decisions too early
- is the product introducing itself clearly enough

## 6. Final summary

Summarize:
- activation target
- flow structure improvements
- friction reduced
- states and transitions handled
- remaining drop-off risks if any

# Quality bar

The onboarding flow should feel:
- guided
- low-friction
- premium
- confidence-building
- fast toward first value
- clear without being verbose
