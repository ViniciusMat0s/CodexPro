# Review Checklist

## Correctness
- Does the code appear logically correct?
- Are important edge cases handled?
- Are assumptions explicit?

## Maintainability
- Does it fit repository patterns?
- Is complexity reasonable?
- Is the abstraction level appropriate?
- Is there unnecessary duplication?

## Validation
- Are inputs validated?
- Are contracts explicit?
- Are failure paths clear?

## Testing
- Are critical behaviors protected?
- Is regression risk addressed?
- Are meaningful gaps left open?

## UI/UX
- Are loading/error/empty states handled?
- Is the hierarchy clear?
- Does the interface feel polished enough?
- Is the primary action obvious?

## Accessibility
- Semantics
- labels
- focus states
- contrast
- interaction clarity

## Security
- auth/authz
- secret exposure
- trust boundaries
- unsafe assumptions
