# Dashboard Heuristics

## Information priority
- Does the dashboard emphasize the most important information first?
- Is critical status visually prominent?

## Widget balance
- Are all widgets competing equally?
- Is there a clear visual hierarchy?

## Scanability
- Can a user understand the dashboard quickly?
- Are tables/cards legible and well-grouped?

## Operational feel
- Does the dashboard feel like a working product surface?
- Or just a decorative grid of cards?

## States
- Empty/loading/error states are handled
- Actions remain discoverable
- Filters/search/sorting feel integrated if present
