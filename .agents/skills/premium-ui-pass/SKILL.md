# Purpose

Use this skill when the task is to improve the quality of a user-facing interface in a modern web product.

This skill exists to raise perceived quality.
It is not just for styling.
It is for:
- hierarchy
- composition
- visual rhythm
- UX clarity
- responsive quality
- state polish
- consistency with a premium product standard

# Use this skill when

Use this skill when the request involves:
- refining a dashboard
- improving a settings/account area
- polishing a form or transactional flow
- making a page feel premium
- improving spacing, hierarchy, or composition
- making landing pages look more intentional and conversion-aware
- fixing UI that feels generic, cramped, flat, or unfinished

# Do not use this skill when

Do not use this skill for:
- backend-only work
- architecture-only decisions
- test-only work
- tiny style nits with no meaningful compositional effect
- bug investigations with no UI quality component

# Premium product standard

Default target qualities:
- strong visual hierarchy
- deliberate spacing
- balanced density
- tasteful typography
- restrained use of color
- clean cards and panels
- trustworthy forms
- operational dashboards
- polished loading/empty/error states
- clear primary actions
- premium perceived value

# Workflow

## 1. Understand the existing screen

Inspect:
- page structure
- primary and secondary actions
- hierarchy of information
- typography and spacing rhythm
- cards, forms, panels, tables
- current responsive behavior
- current design language used elsewhere in the repo

## 2. Evaluate the interface

Check for:
- weak hierarchy
- cramped spacing
- inconsistent sizing
- weak CTA emphasis
- cluttered card layout
- poor grouping of information
- unfinished secondary states
- template-looking composition
- unbalanced dashboard weight
- low-trust form layout
- poor responsiveness
- accessibility issues

## 3. Refine toward premium quality

Prefer:
- stronger composition
- clearer section rhythm
- better scanability
- cleaner and calmer spacing
- better typography hierarchy
- more intentional grouping
- polished interaction states
- better empty/loading/error states
- responsive layouts that still feel designed

Avoid:
- decorative noise
- unnecessary gradients/effects
- visual clutter
- equal visual weight everywhere
- arbitrary spacing decisions
- novelty at the expense of clarity

## 4. Re-check complete experience

Review:
- desktop quality
- narrower layouts
- hover states
- focus states
- disabled states
- loading states
- empty states
- form trust and usability
- accessibility basics

## 5. Final summary

Summarize:
- what visual or UX problems were improved
- what changed in hierarchy or layout
- what states/responsiveness improved
- remaining polish opportunities if any

# Quality bar

The screen should feel:
- premium
- intentional
- visually balanced
- modern
- trustworthy
- clean
- product-grade rather than template-grade
