# Purpose

Use this skill for admin and internal backoffice product surfaces.

This skill is for:
- admin dashboards
- management tables
- filters and bulk actions
- CRUD backoffice interfaces
- review and moderation panels
- internal operational tools

# Use this skill when

Use this skill when the request involves:
- creating an admin page
- refining an internal management surface
- improving tables, filters, bulk actions, or record detail flows
- making admin UX clearer and safer
- improving operational density without visual chaos

# Do not use this skill when

Do not use this skill for:
- public marketing pages
- auth entry flows
- pure design-system work
- tiny UI nits with no backoffice significance

# Admin philosophy

Admin surfaces are not marketing UI.
They should optimize for:
- speed of operation
- high scanability
- safe actions
- clarity under data density
- efficient workflows
- confidence in destructive actions

# Workflow

## 1. Define the operational task

Determine:
- what the operator needs to do
- what information they scan most often
- what actions carry risk
- what filters or search matter
- what bulk actions exist
- what details need fast access

## 2. Structure for operational efficiency

Prioritize:
- filters/search near usage
- clear table hierarchy
- visible status and action affordances
- rapid navigation to details
- strong empty/loading/error handling
- safe destructive-action UX

## 3. Review action safety

Check:
- confirmation for destructive actions
- disabled states where appropriate
- permission visibility
- clear result feedback
- reversible actions where possible

## 4. Optimize density without chaos

Prefer:
- compact but readable layouts
- consistent table rhythm
- meaningful status indicators
- predictable row actions
- clear grouping in detail panels

Avoid:
- decorative waste
- cluttered action bars
- ambiguous statuses
- cramped unreadable tables

## 5. Final summary

Summarize:
- admin workflow improved
- tables/filters/actions changed
- safety considerations handled
- remaining operational friction if any

# Quality bar

The admin surface should feel:
- fast
- clear
- powerful
- safe
- high-density but readable
- operationally mature
