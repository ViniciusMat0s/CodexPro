# Auth Flow Checklist

## Entry
- Clear purpose
- Low-friction fields
- Strong CTA clarity

## Validation
- Required fields explicit
- Invalid input handled cleanly
- Client/server expectations aligned

## Edge cases
- invalid credentials
- expired token
- invalid token
- already verified account
- resend verification/reset
- locked or restricted access states if applicable

## Post-submit
- Success path clear
- Failure path clear
- Redirect behavior understandable
