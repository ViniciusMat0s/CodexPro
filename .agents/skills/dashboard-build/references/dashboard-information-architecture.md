# Dashboard Information Architecture

## Primary zone
Show the most important metrics or statuses first.

Ask:
- What must the user understand within 3 seconds?
- What would justify opening this page daily?

## Secondary zone
Support interpretation:
- trends
- recent activity
- secondary summaries
- relevant segmentation

## Action zone
Make important actions easy to find:
- create
- review
- resolve
- export
- filter
- investigate

## Deep data zone
Tables and detailed breakdowns belong after the top-level story is clear.

## Anti-patterns
- equal weight for all widgets
- too many cards above the fold
- decorative stats with no action value
- unreadable density
- no path from insight to action
