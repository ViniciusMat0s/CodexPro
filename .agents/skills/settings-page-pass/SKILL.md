# Purpose

Use this skill for settings, preferences, account, workspace, and configuration pages.

This skill is for:
- account settings
- workspace settings
- profile settings
- notification preferences
- billing settings
- security settings
- API key/settings areas
- configuration-heavy product pages

# Use this skill when

Use this skill when the request involves:
- creating or refining settings pages
- improving settings UX and structure
- making preferences/configuration pages feel more premium
- organizing sections and destructive actions better
- improving clarity in account-management surfaces

# Do not use this skill when

Do not use this skill for:
- public landing pages
- dashboard overview pages
- isolated forms with no settings context
- auth entry flows

# Settings philosophy

Settings pages are trust surfaces.
They should feel:
- calm
- structured
- clear
- safe
- unsurprising
- efficient

The user should always understand:
- what this setting controls
- what happens if it changes
- whether the action is reversible
- which actions are sensitive or destructive

# Workflow

## 1. Define settings information architecture

Group settings intentionally, for example:
- profile/account
- workspace/team
- notifications
- billing
- security
- integrations
- dangerous/destructive actions

Ensure grouping matches user mental models.

## 2. Improve page structure

Prefer:
- calm sectioning
- clear labels and helper text
- good action hierarchy
- obvious save/apply states
- separation of destructive actions
- strong visual trust

## 3. Review setting behavior

Check:
- default value clarity
- unsaved-change behavior
- success/failure feedback
- disabled states
- permission-based visibility
- destructive action safety
- confirmation flows where appropriate

## 4. Improve trust and usability

Ensure:
- settings do not feel cluttered
- actions are not ambiguous
- inputs are easy to scan
- helper text is concise and useful
- the page remains readable under many options

## 5. Final summary

Summarize:
- settings architecture improved
- trust or safety improvements
- forms/actions refined
- destructive or sensitive areas clarified
- remaining confusion risk if any

# Quality bar

The settings experience should feel:
- premium
- calm
- trustworthy
- organized
- safe for important changes
- operationally clear
